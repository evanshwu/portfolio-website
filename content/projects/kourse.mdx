## The problem

University course portals are built for the desktop web, but students live on
their phones. Checking announcements, downloading materials, or asking a
question about an assignment meant opening a laptop and logging into a clunky
site. For my course final project, I wanted that everyday flow to fit in a
pocket.

## What I built

Kourse is an Android app backed by a PHP/MySQL server. The app organizes
everything into three tabs — a course **bulletin** for announcements, a
**files** section for materials, and a **Q&A** discussion board where students
post questions. Each tab is a simple list; tapping a row opens the full detail.
On the server side I built a PHP/MySQL back-end plus a lightweight admin web
page for editing any record in the database directly.

## Tech

An Android front-end talking to a PHP/MySQL back-end, with a browser-based
admin panel for content management.

## Outcome

Kourse was the final project for a university course, delivering a working
end-to-end app — from the Android client through the PHP API to the MySQL
database — with a functioning presentation and Q&A flow that classmates used
during the term.
